# Issues and Resolutions

## Issues Identified
- **Lack of comments in the code**  
- **Single Responsibility Principle (SRP) violated**  
- **DRY (Don't Repeat Yourself) not applied**  
- **UI issues**:
  - No feedback messages on saved notes  
  - Errors not handled clearly (e.g., empty fields, invalid credentials)  
  - Inconsistent UI element sizes and styles across the app and languages

## Resolutions
- **UI Problems**  
  - Added feedback messages for most user interactions to improve clarity  
  - Fixed contrast issues for better accessibility  
  - Ensured consistent element sizes and styles across all languages  

- **Code Problems**  
  - Added comments to improve readability and maintainability  
  - Refactored code to better follow the Single Responsibility Principle (SRP)  
  - Applied DRY principles to reduce duplication  
  - Improved overall code readability and structure
